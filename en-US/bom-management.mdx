## Create Your First BOM

<Steps>
<Step title="Enter BOM Management">
  Click the "BOM Service" button in the left navigation bar
  
  <Info>
  First-time users will see an empty BOM list
  </Info>
</Step>

<Step title="Create New BOM">
  Click "New BOM" button and fill in basic information: - **BOM Name**: e.g.,
  "Arduino Project V1.0" - **Project Description**: Briefly describe project
  purpose - **Version Number**: e.g., "1.0.0" - **Creator**: Your name
</Step>

<Step title="Add Components">
  Add components through the following methods: - **Search & Add**: Search and
  select components - **Manual Add**: Directly input model and parameters -
  **Import BOM**: Import existing BOM from file - **Add from Favorites**: Select
  from favorites
</Step>

<Step title="Set Quantities">
  For each component, set: - **Required Quantity**: Project needs - **Safety
  Stock**: Recommended backup quantity - **Notes**: Special instructions or
  requirements
</Step>

<Step title="Save BOM">
  Click "Save" button to complete BOM creation
  
  <Check>
  BOM successfully created, you can edit and update anytime
  </Check>
</Step>
</Steps>

## Create and Manage BOM

### Create New BOM

**Fill in Basic Information:**

- **BOM Name**: Clearly describe project name
- **Project Description**: Detailed project purpose
- **Version Number**: Follow version naming conventions
- **Creator**: Responsible person information

**Choose Creation Method:**

- **Blank BOM**: Create from scratch
- **Template Creation**: Use existing templates
- **Import Creation**: Import from file
- **Copy Creation**: Based on existing BOM

**Set Access Permissions:**

- **Private**: Only creator can access
- **Team**: Team members can access
- **Public**: Everyone can view
- **Custom**: Custom permission settings

#### Component Numbering Standards

<CodeGroup>
```text Recommended Numbering Format
R1, R2, R3... (resistors)
C1, C2, C3... (capacitors)
L1, L2, L3... (inductors)
U1, U2, U3... (integrated circuits)
Q1, Q2, Q3... (transistors)
D1, D2, D3... (diodes)
J1, J2, J3... (connectors)
```

```text Numbering Rules
- Number by function group
- Number by position within same function
- Reserve expansion space
- Maintain numbering continuity
```

</CodeGroup>

## Version Control

### Version Management Strategy

<Info>
  Version control is an important part of BOM management, ensuring traceability
  of changes
</Info>

#### Version Number Standards

<CodeGroup>
```text Semantic Versioning
Major.Minor.Patch
Example: 1.0.0, 1.1.0, 1.1.1

Major: Major feature changes
Minor: New features, backward compatible
Patch: Bug fixes, backward compatible

````

```text Version Naming
- Use clear, descriptive names
- Include date if needed
- Document changes in each version
- Maintain version history
````

</CodeGroup>
