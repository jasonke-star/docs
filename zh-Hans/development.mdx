<Info>
  **先决条件**： - Node.js 版本 19 或更高 - 包含 `docs.json` 文件的文档仓库
</Info>

按照以下步骤在您的操作系统上安装和运行 Mintlify。

<Steps>
<Step title="安装 Mintlify CLI">

```bash
npm i -g mintlify
```

</Step>

<Step title="本地预览">

导航到包含 `docs.json` 文件的文档目录，并运行以下命令：

```bash
mintlify dev
```

您的文档的本地预览将在 `http://localhost:3000` 可用。

</Step>
</Steps>

## 自定义端口

默认情况下，Mintlify 使用端口 3000。您可以使用 `--port` 标志自定义 Mintlify 运行的端口。例如，要在端口 3333 上运行 Mintlify，请使用此命令：

```bash
mintlify dev --port 3333
```

如果您尝试在已使用的端口上运行 Mintlify，它将使用下一个可用端口：

```md
端口 3000 已被使用。尝试使用 3001。
```

## Mintlify 版本

请注意，每个 CLI 版本都与特定版本的 Mintlify 关联。如果您的本地预览与生产版本不一致，请更新 CLI：

```bash
npm update -g mintlify
```

## 验证链接

CLI 可以帮助验证文档中的链接。要识别任何损坏的链接，请使用以下命令：

```bash
mintlify broken-links
```

## 代码格式化

我们建议在您的 IDE 上使用扩展来识别和格式化 MDX。如果您是 VSCode 用户，请考虑使用 [MDX VSCode 扩展](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) 进行语法高亮，以及 [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) 进行代码格式化。

## 故障排除

<AccordionGroup>
  <Accordion title='错误：无法使用 darwin-arm64 运行时加载 "sharp" 模块'>

    这可能是由于过时的 node 版本。请尝试以下操作：
    1. 删除当前安装的 CLI 版本：`npm remove -g mintlify`
    2. 升级到 Node v19 或更高版本。
    3. 重新安装 CLI：`npm i -g mintlify`

  </Accordion>

  <Accordion title="问题：遇到未知错误">
  
    解决方案：转到设备的根目录并删除 `~/.mintlify` 文件夹。然后再次运行 `mintlify dev`。
  </Accordion>
</AccordionGroup>

想了解最新 CLI 版本的变化？查看 [CLI 更新日志](https://www.npmjs.com/package/mintlify?activeTab=versions)。
